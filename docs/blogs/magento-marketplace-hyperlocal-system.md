# Magento Marketplace Hyperlocal System

Updated 17 October 2017

[Buy Now](https://store.webkul.com/Marketplace-Hyperlocal-System-Magento.html)

**Marketplace Hyperlocal System** is very beneficial for both sellers and buyers along with the admin. The Seller/admin can easily add the location where they can ship their products. Customers can enter their location and check the available products and sellers in that location. Sellers and the admin can add their location according to which their products are displayed to the customer. The seller and the admin can also add shipping rate according to the location range.

Magento Marketplace Hyperlocal System module is an add-on of [Magento Marketplace](https://store.webkul.com/Magento2-Marketplace.html) module, you need to install Magento Marketplace first to use this module.

This Module is also available for **[Magento2](https://store.webkul.com/Marketplace-Hyperlocal-System-Magento2.html).**

****Note**: You must have the **Google API Key** to make use of this extension. To generate one for you **[click here](https://developers.google.com/maps/documentation/javascript/get-api-key)**.**  

**

### Features Of Magento Marketplace Hyperlocal System

- The customer can enter their location and buy products from nearest seller’s store.
- The customer can also change their location.
- The customer can check seller’s availability in their location.
- The customer can check product’s availability in their location.
- The seller can add multiple locations.
- The seller can add Hyperlocal table rate shipping.
- The seller can add shipping price using CSV file.
- Admin can also add multiple locations.
- Admin can add shipping on behalf of sellers using a CSV file.
- Admin can add radius unit and range which will be used to search the seller from customer’s location.
- If the customer does not select his location then his location will be the location that is set by the admin.

### Admin Management – Magento Marketplace Hyperlocal System

After installation of the module, admin can configure the module by going to **System > Configuration > Webkul > Hyperlocal Marketplace**. Here admin has to enter the heading for location popup window. Admin can select the radius unit and enter the radius range which is used to search sellers from customers location. Admin has to provide the Google API Key so that Google search suggestion for location will work.

[!\[Configuration Settings\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/1-config..png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/1-config..png)

**Note:** For Google API Key please visit [https://developers.google.com/maps/documentation/javascript/get-api-key](https://developers.google.com/maps/documentation/javascript/get-api-key)

Now the admin can configure the Hyperlocal Shipping from **System > Configuration > Sales > Shipping Methods > Hyper Local Shipping. **From here admin can enable or disable the shipping method and can set the Shipping Method name that will be visible on front-end.

[!\[Hyper Local Shipping\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/2-shipping.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/2-shipping.png)

Admin can add Ship Areas and Ship Rates by going to **Marketplace Management > Hyper Local**.

On clicking **Ship Area**, admin can view and manage all the added Shipping areas. Admin can add/edit or delete any ship area from here.

[!\[Ship Area\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/3-ship-area.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/3-ship-area.png)

On clicking “**Add Ship Area**” a new page opens up where the admin has to enter the location and then the latitude and longitude will automatically get filled. Then admin has to save the new ship area.

[!\[Add Ship Area\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/4-add-ship-area.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/4-add-ship-area.png)

Admin can see and manage the Shipping Rates from here. Admin can also delete any ship rate from here and can also add new ship rates.

[!\[Ship Rate\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/5-ship-rate.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/5-ship-rate.png)

Admin can download the sample CSV file and can then upload the CSV file containing the new ship rates.

[!\[Add Ship Rate\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/6-add-ship-rate.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/6-add-ship-rate.png)

### Seller Management – Magento Marketplace Hyperlocal System

After the module is installed, sellers can see the “**Hyper Local**” option in marketplace menu. After clicking on the ‘Hyper Local’ seller will find two options “**Add Shipping Area**” and “**Ship Rate**“.

Sellers can click on ‘Add Shipping Area’ to add the locations where they can ship their products. Here seller can add locations and see the list of locations which is already added. Seller can also delete any location if they want.

[!\[Add Ship Area - Seller\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/7-seller-ship-area.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/7-seller-ship-area.png)

Seller can add shipping rates by clicking on “**Ship Rate**” and from here seller can upload the CSV file of new Shipping Rates. Seller can also delete any previously added rate.

[!\[Add Ship Rate - Seller\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/8-seller-ship-rate.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/8-seller-ship-rate.png)

### Customer Management – Magento Marketplace Hyperlocal System

When a customer will visit the website then a pop-up will open asking to enter the location. The customers can enter their location in the pop-up and click on the “**Go To Shop**” button to save the location.

[!\[Front-end\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/9-customer.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/9-customer.png)

On changing the location the customer will see a pop-up notification stating “On location address change cart will empty”. So if there is any product in the cart then it will get removed from the cart and cart will become empty on location change.

[!\[Warning message\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/10-msg.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/10-msg.png)

If no product is available in the selected location for the selected category than the following message will be shown.

[!\[No product available\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/11-no-products.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/11-no-products.png)

If the products are available for the selected location in the selected category then the customer will be able to see and purchase them.

[!\[Available Products\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/12-products.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/12-products.png)

While purchasing a product, the customer can select the Hyper Local Shipping Method.

[!\[Hyper Local Shipping - Checkout\](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/13-checkout.png)](http://cdnblog.webkul.com/blog/wp-content/uploads/2017/01/13-checkout.png)
