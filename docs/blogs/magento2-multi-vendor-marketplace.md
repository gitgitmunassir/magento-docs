# User Guide for Adobe Commerce Multi Vendor Extension

Updated 7 May 2026

[Live Demo](https://magento2demo.webkul.in/marketplace/)

[Magento 2 Multi Vendor Extension](https://store.webkul.com/magento2-multi-vendor-marketplace.html) provides a platform where multiple sellers (vendors) can register to marketplace, list, and sell their products or services to customers.

Adobe Commerce Marketplace Extension converts your online store into a complete online marketplace shop.

Therefore, using Magento 2(Adobe Commerce) Marketplace Extension, the seller can manage the inventory, shipment, Seller Profile page, Seller Collection page, and much more.

The seller can add multiple product types like Simple, Configurable, Virtual & Downloadable.

Furthermore, Magento 2 Marketplace Plugin supports all the Magento 2 templates and themes and it is compatible with RWD ([responsive web designs](https://webkul.com/website-development/)).

If you’re looking to make your customers as resellers on your Magento 2 store, then you can check[ Magento 2 Reseller Extension](https://store.webkul.com/magento2-reseller-extension.html)

**Important Notes: **

1. If you’re using Magento version 2.3.0-2.3.3, then please install the Patch file v2.3.0-2.3.3 available in the Magento 2 Marketplace plugin zip.
1. There is no need to install the patch for versions above Magento v2.3.3.
1. The greater version of the Marketplace Extension will not work with the lower version of Magento.
1. If you have Magento 2.0.x then you can use a lower version of the Marketplace.
1. The Multi-Vendor Marketplace version 2.0.1 will not work with the Magento 2.0.x version, it will work with Magento 2.1.x.
1. To manage the Multi-Source Inventory feature at the sellers’ end. You must have installed Marketplace MSI for Magento 2.

### Features of Magento 2 Marketplace Extension

- Magento 2 multi vendor module is compatible with the Hyva Theme.

**Note:** Hyva module currently supports only the default vendor dashboard.

### Advance Admin End Features

- Easy configuration of the Marketplace Module.
- Also, magento 2 multi vendor module offers an Attractive design for managing the Products, Sellers, and commissions.
- Additionally, this module provides an option to view the list of all the sellers in the marketplace in one place.

- Furthermore, the admin can set the status of the seller profiles as Approve/Disapprove/Deny/Processing
- Even, the admin can allow a separate vendor dashboard.
- Admin can manage the Seller profile page setting.
- Admin can assign a category and subcategory to the seller.
- Further, the Admin can view, approve, disapprove, update, deny, and delete a seller-requested product.
- Admin can enable/disable minimum order functionality.
- Admin can set minimum order amount.
- The admin can Inline edit the products in the admin end grid.
- Admin can send global news to sellers.
- Admin can manage the fulfillment label, text and image.

### Other Features of Admin End

- Additionally, the admin can view the list of unapproved seller products.
- Plus, the admin can approve mass products and sellers at a time.
- The Admin can allow a seller to add a product limit for a customer on product purchases.
- Furthermore, the admin can allow a seller to add related products, up-sell products, and cross-sell products from the seller’s panel.
- Plus, the seller gets a mail notification when product inventory reaches the specified minimum level set with the admin.
- Also, the seller can view hints for fields if enabled and the admin can set them.
- Configuration setting for SKU prefix that the admin can configure for seller’s products.
- In addition, the admin can view notifications for seller requests, seller products, and feedback reviews.
- The admin can create email custom templates too.

- Also, the admin can configure marketplace landing page options like- banner, banner content, button label, icons, etc
- In addition, the admin can choose to auto-approve orders or not.
- Also, the admin can view the Product Preview for product approval.
- The admin can enable/disable separate dashboard seller wise.
- Additionally, features a Product Assignment tab to render the product grid to mass assign products to the seller in the seller edit page in the admin panel.

- Admin can now refine the search process of enabling the seller filter in layer navigation.

When the module zip extracts, follow path src>app and then copy the app folder into the Magento 2 root directory on the server as shown below:

```

```

```
php bin/magento setup:di:compile
```

```
php bin/magento setup:static-content:deploy
```

```
php bin/magento indexer:reindex
```

```
php bin/magento cache:flush
```


### Magento 2 Marketplace Extension Configuration

### General Settings – Magento 2 Multi Vendor Extension

Here, the admin can set –

- Admin Name – Firstly the admin needs to enter his name.
- Admin Email-ID – After that In this field, the admin will enter their email address.
- Global Commission Rate – Then, in the global commission, the admin needs to enter a global commission rate in percentage, this commission mechanism is applicable to all the sellers.

Additionally, let us say as an example there is a seller S1 and his/her product(P1) price is 100 USD.

So on the sale of product P1 seller will get 80 USD and the admin will get 20 USD as commission if the commission is configured as 20%.

- Deduct Discount Amount From – With this feature the admin can set the deduction of the discount amount, whether it will be deducted from the seller’s total amount or admin commission amount, or both.
- Allow Seller to Manage Orders – By enabling “YES”, the admin allows the sellers to manage the orders from the front end.

Now, the seller can generate invoices for their order products, generate a tracking number, and shipment for the orders of their products.

In addition,  the seller can notify the customers by sending them emails, cancel/refund the order, and even print the order.

- Move Product Tax to Seller Account – Thus, most importantly, If this option is disabled then, the product Tax amount will forward into the Admin account otherwise, it will go into the Seller account.
- Seller Approval Required – Likewise, partner or seller approval is the same as product approval, during sign-up of the seller if the admin allows this option.

Furthermore, then the user who will sign up in the store as a seller will convert into the seller automatically. The store owner can set **YES or NO**.

### **Set Captcha Enable** – 

If this is set as “Yes” then if any user contacts the seller then on the Contact Form, the additional question appears for the security purpose.

Furthermore, if the option is set as “No” then the customer can contact the sellers without any additional security questions.

- Allow Seller for Google Analytics – The admin can select “Yes” to let the seller use google analytics.

- Image –  Upload the image by selecting them and will be shown on product page which will be fulfill by admin.
- Label – Write the name and will be used to show in fulfillment label at admin and seller product edit page.
- Text – Write a text and will shown on product page.

### How to Use Advanced Pricing in Multi-Vendor Marketplace?

Advanced pricing can be said as the special price offered to a specific group of customers. Also, advanced pricing can be implemented on simple, virtual, downloadable, and bundle products.

And Advanced pricing data is synchronized with product pages. For example, if you update a tier price quantity, the system updates the value on the product page.

- On the admin end, the advance pricing can be set on the product. For this,
- The admin needs to open the product in edit mode.
- After which under Price we have Advanced Price, where we can set
  - Special Price(along with the date of validity)
  - Tier Price

## How To Make Tier Price Visible On Seller End?

Certainly, the Tier Price is a major component of Advance Pricing which gives the customer the advantage of receiving a discount on the basis of the quantity purchased.

That is, you offer a quantity discount on a product listing or product page in the storefront. However, the discount can be applied to a specific store view or customer group.

![tier-price-frontend
](https://cdnblog.webkul.com/blog/wp-content/uploads/2024/06/tier-price-frontend-image-1.webp)

**Extended Functionality-** The Magento 2 Marketplace Extension is having an additional feature.

That enables the seller to have a facility to set tier price for the product for a specific group of people too.

Moreover, this achieves by[ Magento 2 Custom Attribute Marketplace Add-on](https://store.webkul.com/Magento2-Marketplace-Custom-Attribute.html), we can simply go to **Custom Attributes **in the admin panel.

Also, the admin can select the ‘tier_price’ attribute and under the **Action** drop-down, click on ‘**Show In Front**.

![tier price ](https://cdnblog.webkul.com/blog/wp-content/uploads/2024/06/tier-price-image-1-1-1200x528.webp)

Now, the attribute **Tier Price** could be seen on seller front when we open the product in edit mode.

![add-advance-pricing](https://cdnblog.webkul.com/blog/wp-content/uploads/2015/12/add-advance-pricing.webp)
